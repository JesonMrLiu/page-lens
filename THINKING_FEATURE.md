# 思考推理功能说明

## 功能概述

PageLens 现已支持**思考推理**功能，通过多轮 API 调用实现深度思考，帮助 AI 更好地理解和回答复杂问题。

## 思考模式

| 模式 | 说明 | 默认轮数 |
|------|------|----------|
| 直接回答 | 不使用思考功能，直接回答问题 | - |
| 一般思考 | 进行1轮思考后给出答案 | 1轮 |
| 深度思考 | 进行多轮深度思考后给出答案 | 3轮 |

## 使用方法

### 1. 选择思考模式

在聊天输入框上方，可以看到三个思考模式按钮：

- **直接回答**：适合简单问题，快速获得答案
- **一般思考**：适合需要一定分析的问题
- **深度思考**：适合复杂问题，需要深入分析

### 2. 发送消息

选择思考模式后，输入问题并发送。AI 会：

1. **思考阶段**：根据选择的模式进行多轮思考
   - 每轮思考会基于前几轮的结果进行更深入的分析
   - 思考过程会实时显示在消息中

2. **总结阶段**：基于所有思考结果给出最终答案

### 3. 查看思考过程

在 AI 回答的消息中，可以看到：

- **思考过程面板**：点击展开查看每轮思考的详细内容
- **最终回答**：基于思考结果的完整答案

## 配置思考轮数

在**设置 > 通用**页面，可以配置：

- **一般思考轮数**：1-5轮（默认1轮）
- **深度思考轮数**：1-5轮（默认3轮）

更多轮数可以提供更深入的分析，但会消耗更多时间和 Token。

## 适用场景

| 场景 | 推荐模式 |
|------|----------|
| 简单问答 | 直接回答 |
| 代码解释 | 一般思考 |
| 问题分析 | 一般思考 |
| 方案设计 | 深度思考 |
| 复杂推理 | 深度思考 |
| 多因素决策 | 深度思考 |

## 技术实现

### 工作流程

1. 用户选择思考模式并发送消息
2. Background Service Worker 接收请求
3. 根据思考轮数进行多轮 API 调用：
   - 每轮调用会包含前几轮的思考结果作为上下文
   - 思考指令引导 AI 进行深入分析
4. 所有思考轮次完成后，进行一次总结调用
5. 将思考过程和最终答案返回给用户

### 消息流程

```
用户消息
    ↓
[第1轮思考] → 思考结果1
    ↓
[第2轮思考] + 思考结果1 → 思考结果2
    ↓
[第3轮思考] + 思考结果1,2 → 思考结果3
    ↓
[总结] + 所有思考结果 → 最终答案
```

## 注意事项

1. **Token 消耗**：思考模式会消耗更多 Token，因为需要进行多次 API 调用
2. **响应时间**：思考轮数越多，响应时间越长
3. **模型支持**：所有 OpenAI 兼容的模型都支持此功能
4. **取消操作**：在思考过程中可以随时点击停止按钮取消
